Vietnamese Banking Business Case

Why Loan & Deposit Analytics Are the #1 Priority

Vietnamese commercial banks operate on net interest margin (NIM) — the spread between lending rates and deposit costs. Under tightening SBV credit quotas, post-COVID NPL pressure, and intensifying deposit competition, management needs same-day decision intelligence — not 3-week-old reports.

Loan Portfolio Management

Track disbursement velocity, loan growth, product mix, and maturity schedules. Detect over-concentration in a sector, region, or customer group before it becomes a credit event.

Deposit Mobilization & CASA

CASA ratio is the direct proxy for funding cost. In Vietnamese banks, CASA below 20% is a warning signal — high funding cost directly compresses NIM. Leading banks (VCB, TCB) target 40–50% CASA.

LDR Ratio (TT22)

Per Circular 22/2019/SBV: LDR must not exceed 85% for state-owned banks and 80% for private commercial banks. This dashboard shows LDR intra-month — not 3 weeks after month-end.

DAX Measures — Production Ready

All measures are organized in a dedicated _BankingMeasures table. Use CALCULATE with time-intelligence for all period comparisons.

01 — Total Loan Outstanding
Total Loan Outstanding = SUM( FactDuNoTinDung[DuNoCuoiKy] ) -- Sum of all active outstanding loan balances in the current filter context. -- Responds to all slicers: branch, product, segment, date.
02 — Loan Previous Month
Loan Previous Month = CALCULATE( [Total Loan Outstanding], PREVIOUSMONTH( DimNgay[NgayDayDu] ) ) -- Uses PREVIOUSMONTH to retrieve prior month loan balance. -- Requires DimDate to be marked as a Date Table in Power BI.
03 — Loan Growth %
Loan Growth % = VAR DuNoThangNay = [Total Loan Outstanding] VAR DuNoThangTruoc = [Loan Previous Month] RETURN DIVIDE( DuNoThangNay - DuNoThangTruoc, DuNoThangTruoc, BLANK() ) -- Month-over-month growth rate. Format as percentage. -- Returns BLANK() when no prior month data exists (avoids misleading zeros).
04 — CASA Balance (CASA Balance)
CASA Balance = CALCULATE( SUM( FactHuyDongVon[DepositBalance] ), FactHuyDongVon[CoPhaiKKH] = 1 ) -- Current accounts + demand savings (CASA) (CASA). -- Low-cost funding — key metric for NIM management.
05 — CASA Ratio
CASA Ratio = DIVIDE( [CASA Balance], [Total Deposits], 0 ) -- % of total deposits in low-cost CASA. Target: > 30%. -- Red alert when it falls below 20%. Use conditional formatting.
06 — LDR Ratio (Circular 22/SBV)
LDR Ratio = DIVIDE( [Total Loan Outstanding], [Total Deposits], 0 ) -- Loan-to-Deposit Ratio. Per Circular 22/2019/SBV: -- ≤ 85% (state-owned banks) or ≤ 80% (private commercial banks). -- Add a reference line at threshold for visual alert.
07 — NPL Balance (NPL — Group 3+4+5)
NPL Balance = CALCULATE( SUM( FactDuNoTinDung[DuNoCuoiKy] ), FactDuNoTinDung[NhomNo] >= 3 ) -- NPL = Group 3 (Substandard) + Group 4 (Doubtful) + Group 5 (Loss). -- Per Circular 02/2013/SBV (amended by Circular 03/2023).
08 — NPL Ratio
NPL Ratio = DIVIDE( [NPL Balance], [Total Loan Outstanding], 0 ) -- Core risk metric. SBV alert threshold: 3%. -- Color-code: Green < 2% | Amber 2–3% | Red > 3%.
09 — Net Interest Income (NII)
Net Interest Income = VAR ThuLai = SUMX( FactDuNoTinDung, FactDuNoTinDung[DuNoCuoiKy] * FactDuNoTinDung[LaiSuatVay] / 12 ) VAR ChiLai = SUMX( FactHuyDongVon, FactHuyDongVon[DepositBalance] * FactHuyDongVon[LaiSuatHuyDong] / 12 ) RETURN ThuLai - ChiLai -- Monthly NII estimate (annual rate / 12). -- For higher accuracy, use actual accrued interest data from CoreBanking.
10 — Provision Coverage Ratio
Provision Coverage Ratio = DIVIDE( SUM( FactNoXau[ProvisionRequired] ), [NPL Balance], 0 ) -- Provision coverage over NPL balance. Target: ≥ 100%. -- Below 100% = under-provisioned, requires additional provision → P&L impact.
11 — Loan Classification by TT02 Group (1–5)
DN Group 1 = CALCULATE([Total Loan Outstanding], FactDuNoTinDung[NhomNo] = 1) DN Group 2 = CALCULATE([Total Loan Outstanding], FactDuNoTinDung[NhomNo] = 2) DN Group 3 = CALCULATE([Total Loan Outstanding], FactDuNoTinDung[NhomNo] = 3) DN Group 4 = CALCULATE([Total Loan Outstanding], FactDuNoTinDung[NhomNo] = 4) DN Group 5 = CALCULATE([Total Loan Outstanding], FactDuNoTinDung[NhomNo] = 5) -- 5 separate measures for the 5 TT02 loan groups. -- Used in stacked bar charts to visualize loan portfolio quality structure.

Portfolio Project · Banking BI · Power BI

Banking Loan & Deposit Analytics Dashboard

A consulting-grade Power BI solution replacing manual Excel reporting with a real-time executive intelligence system for a mid-size Vietnamese commercial bank.

Power BI SQL Server CoreBanking Integration TT02/TT22 Compliance Row-Level Security
Problem Business Loan

A private commercial bank with 80 branches and VND 85,000 Bn in total assets managed all performance reporting manually. The credit team exported from CoreBanking (T24), the deposit team used its own Excel tracker, and the risk team pulled from an internal NPL system. No unified view connected these three critical data streams. The CRO discovered branch NPL threshold breaches at the monthly provision meeting — not through proactive alerts. Branch Directors received static PDFs with no drill-down capability.

Solution Delivered

Designed and deployed a 6-page Power BI Banking Dashboard backed by a multi-fact star schema with 5 Fact tables and 7 Dimension tables. Built a SQL staging layer consolidating data from CoreBanking, credit management, and NPL classification systems — auto-refreshed at 6:00 AM daily. Implemented 3-tier Row-Level Security: Branch → Region → Head Office. Built the 5-group loan classification framework per Circular 02/2013/SBV and LDR monitoring per Circular 22/2019/SBV.

Lessons Learned
  • CoreBanking exports were inconsistent across branches — loan status codes differed per installation. Data standardization consumed 35% of project time.
  • The CRO needed daily NPL data but the system only classifies monthly — solved with a DPD-based proxy NPL calculation from daily balance files.
  • Branch Directors needed a mobile-optimized view — the desktop layout was too wide for tablet use in branch meetings.
  • Row-Level Security must be planned from Day 1, not retrofitted at go-live.
Future Enhancements
